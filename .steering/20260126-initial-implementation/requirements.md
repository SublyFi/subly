# 要求内容

## 概要

User Dashboard（ユーザーダッシュボード）の初回実装。Sublyのサブスクリプションユーザー向けのWebインターフェースを構築する。

## 背景

Sublyプロトコルでは、ユーザーがプライバシーを保護した状態でサブスクリプション管理と資金運用を行う。User DashboardはProtocol A（Devnet: 会員管理）とProtocol B（Mainnet: Vault）の両方を操作するUIを提供する。

初回実装では、SDKがまだ完成していないため、UIの基盤構築とモックデータを使用したプロトタイプを作成する。

## 実装対象の機能

### 1. プロジェクト基盤

- Next.js 16 + React 19 + TailwindCSS 4のセットアップ
- shadcn/uiコンポーネントの導入
- プロジェクト構造の確立（App Router）

### 2. ウォレット接続

- Solana Wallet Adapter統合
- Phantom、Backpackなど主要ウォレット対応
- 接続状態の管理

### 3. 共通レイアウト

- ヘッダー（ロゴ、ナビゲーション、ウォレット接続ボタン）
- サイドバー/タブナビゲーション（Vault、Subscriptions、History）
- フッター

### 4. Vault画面（Protocol B - Mainnet）

- 残高表示（モックデータ）
- 運用状況（APY、累計利回り）
- 入金フォーム（UI のみ）
- 出金フォーム（UI のみ）
- 定期送金設定一覧

### 5. Subscriptions画面（Protocol A - Devnet）

- 契約中のサブスクリプション一覧
- プラン検索・契約UI
- 契約詳細表示
- 解約確認ダイアログ

### 6. History画面

- 取引履歴一覧
- フィルタリング（タイプ、日付）

## 受け入れ条件

### プロジェクト基盤

- [ ] `npm run dev`でローカル開発サーバーが起動する
- [ ] `npm run build`でビルドが成功する
- [ ] TailwindCSSでスタイリングが適用される
- [ ] shadcn/uiコンポーネントが使用可能

### ウォレット接続

- [ ] Phantomウォレットで接続できる
- [ ] 接続・切断が正常に動作する
- [ ] 接続状態がヘッダーに表示される

### 共通レイアウト

- [ ] 全ページで統一されたレイアウトが表示される
- [ ] ナビゲーションでページ遷移できる
- [ ] レスポンシブデザインが適用される

### Vault画面

- [ ] 残高（モックデータ）が表示される
- [ ] 入金フォームが表示され、入力できる
- [ ] 出金フォームが表示され、入力できる
- [ ] 定期送金一覧が表示される

### Subscriptions画面

- [ ] 契約中のサブスクリプション一覧が表示される
- [ ] プラン検索画面が表示される
- [ ] 契約・解約のダイアログが表示される

### History画面

- [ ] 取引履歴一覧が表示される
- [ ] フィルタリングが動作する

## 成功指標

- 全画面のUIがデザインに従って実装されている
- モックデータで一通りの操作フローを確認できる
- 将来のSDK統合に備えた構造になっている

## スコープ外

以下はこのフェーズでは実装しません:

- SDK連携（@subly/membership-sdk、@subly/vault-sdk）
- 実際のブロックチェーントランザクション
- バックエンドAPI連携
- ユニットテスト・E2Eテスト
- 本番環境へのデプロイ
